## 00-overview - 全局架构概览

### 核心概念
- Claude Code = 本地终端里的 AI 编程编排器
- 不是模型本身，而是站在模型和电脑之间
- 主要职责：收集上下文 → 组装 Prompt → 调用模型 → 执行工具 → 权限校验

### 核心组件
```
终端输入 → main.tsx → QueryEngine.ts → query.ts → tools.ts → permissions/* → compact/* → mcp/*
```

### OpenClaw 可以借鉴的
1. **QueryEngine 模式** - 小溪现在直接调用模型，可以借鉴这个分离"请求组装"和"执行"的模式
2. **多层职责分离** - Context 收集、Prompt 组装、工具执行、权限校验分开的架构
